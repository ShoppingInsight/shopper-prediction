KNN 구매 가능성 모델 — 설명 문서 (로컬 전용)

models/knn/knn.ipynb 를 읽기 위한 개인용 정리. git에 커밋되지 않음(.git/info/exclude로 제외).

1. 한 줄 요약

KNN으로 "이 세션이 구매할까?" 를 점수화한다. 다만 잘 보정된 확률 모델이 아니라, 구매 가능성이 높은 세션을 위로 끌어올리는 '우선순위(랭킹) 모델' 로 보는 게 맞다.

2. 데이터

  • 전처리 노트북이 저장한 KNN 전용 split 사용 (models/knn/csv/).
  • 학습 9,764 / 테스트 2,441, 양성(구매) 비율 ≈ 15.6% (불균형).
  • 입력 11개 변수: Administrative, Administrative_Duration, Informational, Informational_Duration, ProductRelated, ProductRelated_Duration, BounceRates, ExitRates, SpecialDay, Weekend, is_new_visitor.
  • PageValues 제외(누수 변수), ID·Month 제외, 중복 제거 후 데이터.
  • 이 CSV는 SMOTE/스케일 적용 전 상태 → 모델 전처리는 Pipeline 안에서 수행.

3. 모델 구조 — 누수 없는 Pipeline

log1p → StandardScaler → SMOTE → KNNRegressor
  • KNN은 거리 기반이라 큰 값이 거리를 지배 → 우편향 변수에 log1p, 이어서 표준화.
  • SMOTE는 Pipeline 안에 둔다: GridSearchCV/CV가 fold마다 Pipeline 전체를 새로 fit하므로 SMOTE가 검증 fold를 보지 못한다(오버샘플링 누수 방지). 이게 이 노트북의 핵심 설계.
  • KNeighborsRegressor로 이웃의 평균(=구매 비율)을 점수로 사용.

4. 실험 흐름

단계 내용
raw baseline log1p/스케일/SMOTE 없이 원본 거리 KNN(K=5) — 비교 기준선
수동 K 탐색 StratifiedKFold로 K만 바꿔 ROC-AUC/PR-AUC/Brier 비교
GridSearchCV K × 거리(p=1 Manhattan / p=2 Euclidean) × weights(uniform/distance) 동시 탐색
최종 재학습 CV 1위 조합을 전체 train에 fit 후 test 평가

최종 선택: K=151, weights=distance, p=1 (Manhattan), CV ROC-AUC ≈ 0.7316.

5. 테스트 성능 (raw baseline → 최종 Pipeline)

📖 지표 용어 풀이 — 따로 찾아볼 필요 없이 한 줄 직관.
  • ROC-AUC — 랜덤 구매자가 랜덤 비구매자보다 높은 점수를 받을 확률(=순위 능력). 0.5=찍기, 1=완벽, 높을수록 좋음. 클래스 불균형에 둔감.
  • PR-AUC(Average Precision) — Precision-Recall 곡선 아래 면적. 구매자 15.6%뿐인 불균형에서 ROC보다 정직. 무정보 기준선이 곧 양성비율(0.156)이라 0.299면 약 2배. 높을수록 좋음.
  • Brier — 평균((예측확률 − 실제)²). 예측값을 진짜 확률로 믿어도 되나(보정)를 봄. 낮을수록 좋음. (나빠지면 "0.56=구매확률 56%"처럼 읽으면 안 됨)
  • Precision — 산다고 예측한 것 중 진짜 구매 비율(오탐 관리). 높을수록 좋음.
  • Recall — 진짜 구매자 중 잡아낸 비율(놓침 관리). 높을수록 좋음. Precision과 트레이드오프.
  • F1 — Precision·Recall의 조화평균(균형). 한쪽만 높으면 같이 낮아짐. 높을수록 좋음.
지표 raw baseline 최종 방향
ROC-AUC 0.6173 0.7284 ↑ 약 +18%
PR-AUC 0.2084 0.2988 ↑ 약 +43% (불균형에서 중요)
Recall 0.0995 0.8246 ↑ 크게
Precision 0.3016 0.2427 ↓ (오탐 증가)
F1 0.1496 0.3750 ↑
Brier 0.1438 0.2352 ↓ 나빠짐(확률 보정 X)

→ 순위화 성능은 확실히 좋아졌지만, 확률값 자체의 신뢰도(Brier)는 나빠졌다. 구매 후보를 넓게 잡는(Recall↑) 대신 오탐(Precision↓)을 감수한 구조.

6. 인사이트 (노트북 5절)

  • 5-1 Decile: 예측 점수 10구간 중 최하위 구매율 ≈1.2% → 최상위 ≈31.1% (약 25배). 큰 방향은 맞지만 완전 단조는 아님 → 개별 확률값을 그대로 믿으면 안 됨.

- 5-2 상/하위 그룹 행동차: 상위 10%는 ProductRelated_Duration ≈2828 vs 하위 ≈67, ProductRelated ≈72 vs ≈4. 상품 탐색 강도가 가장 강한 구매 신호. 상위 그룹은 ExitRates·BounceRates가 낮음(이탈 적음). - 5-3 Permutation Importance: is_new_visitor > ProductRelated_Duration > ExitRates

ProductRelated 순으로 의존. Informational·Weekend는 음수(거리 계산에서 잡음에 가까움).

- 5-4 이웃 예시: 예측확률 1.0인데 실제 구매 0인 샘플. K=151 중 구매 이웃은 6명뿐인데도 거리 0인 가장 가까운 구매 이웃이 distance 가중치로 점수를 끌어올림 → 확률값의 한계 예시. - 5-5 결론: "누가 반드시 사는가"가 아니라 "어떤 세션을 먼저 볼 것인가" 를 정하는 1차 필터.
