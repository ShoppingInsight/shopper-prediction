K-Means 방문자 세분화 — 설명 문서 (로컬 전용)

models/clustering/clustering.ipynb 를 읽기 위한 개인용 정리. git에 커밋되지 않음(.git/info/exclude로 제외).

1. 한 줄 요약

지도학습 3종(DT/RF/KNN)이 "구매할까?" 를 예측한다면, 이 노트북은 비지도로 "방문자가 행동상 어떤 유형으로 나뉘고, 유형별 구매율이 어떻게 다른가" 를 분석한다.

2. 핵심 원칙 (비지도의 정의 지키기)

  1. Revenue(구매 여부)는 군집 학습에 쓰지 않는다. 순수 행동 변수로만 군집을 만든 뒤, 사후에 실제 구매율을 붙여 해석/검증한다.
  2. KNN 전처리 입력을 그대로 재사용한다(K-Means도 유클리드 거리 기반, 요구사항 동일). ID·Month 제외, 11개 수치 컬럼, 누수 변수 PageValues 제외.
  3. 군집은 기술적(descriptive) 분석 → train/test 합친 전체 세션(중복 제거 후 12,205건)에 적용.

3. 전처리

log1p (우편향 9개 컬럼) → StandardScaler
  • Weekend, is_new_visitor는 이미 0/1이라 log 생략.
  • Revenue 미포함(비지도 유지).

변수(컬럼) 한글 풀이

군집에 쓰는 11개 행동 변수(PageValues·Month·ID성 변수 제외).

4. K 선택 — 엘보우 + 실루엣

  • 엘보우(inertia): K=2→3에서 가장 크게 꺾인 뒤 완만 → 굴절점 ≈ 3.
  • 실루엣: K=3에서 ≈0.32의 또렷한 국소 최댓값. (K=7이 0.327로 미세하게 높지만 해석만 복잡)
  • 해석 가능성: K=3이 즉시이탈 / 일반탐색 / 고관여 3단계로 깔끔, 구매율 단조 증가.
  • → K=3 채택. (자동 argmax가 아니라 3지표 종합 판단)

5. 3개 군집 프로파일

cluster 라벨 규모 구매율 행동 특징
0 Browsers (일반 탐색) ~74% (8,988) ~15% 상품 ~26페이지, 평균적 탐색, 다수
1 Engaged (고관여) ~19% (2,313) ~24% 상품 ~66페이지·체류 ~2,600초, Admin 페이지도 많음
2 Bouncers (즉시 이탈) ~7% (904) ~0.4% 상품 ~3페이지·~40초, Bounce/Exit 매우 높음

핵심 인사이트: 세션 깊이가 깊어질수록 구매율이 단조 상승한다 (0.4% → 15% → 24%). 이탈률 높은 군집은 거의 구매하지 않는다.

주의: 구매율(rate) ≠ 구매자 수(count)

차트가 보여주는 건 구매율(비율) 이지 구매자 수가 아니다. 둘은 1등이 다르다.

군집 세션 수 구매율 실제 구매자 수 전체 구매자 중 비중
0 Browsers 8,988 (74%) 15.0% 1,350 70.8%
1 Engaged 2,313 (19%) 24.0% 554 29.0%
2 Bouncers 904 (7%) 0.4% 4 0.2%

  • 구매율 1등 = 군집 1(24%), 구매자 수 1등 = 군집 0(1,350명, 전체 구매자의 71%). 군집 0이 인원이 4배 많아 "사는 비율"은 평범해도 실제 산 사람 수는 압도적이다.
  • 군집 2가 거의 안 사는 건 인원이 적어서가 아니다. 구매율은 이미 인원수로 나눈 값이라 크기와 무관하다. 904명 중 4명만 샀고, 인원이 9,000명이었어도 비율은 ~0.4%였을 것 — 행동 패턴 자체가 안 사는 유형(즉시 이탈)이라는 뜻. 인원이 적은 것과 구매율이 0에 가까운 건 별개 사실.
  • 군집 0 해석: 전체의 74%인 평균적 다수. 구매율 15%는 전체 평균(15.6%)과 거의 같지만, 규모가 커서 매출(구매자)의 71%가 여기서 나온다. 비율은 평범해도 절대 인원이 커 전환 개선 여지가 가장 큼 (15%→17%로 2%p만 올려도 구매자 ~180명 증가 ≈ 군집 1 구매자의 1/3). → 넛지/전환 유도의 핵심 타깃.

한 줄: 군집 1 = 가장 잘 사는 유형(비율), 군집 0 = 가장 많이 사는 유형(머릿수), 군집 2 = 거의 안 사는 유형(비율 자체가 0에 가까움, 인원 탓 아님).

6. 교차 분석 (범주형 신호)

  • 세 군집 모두 재방문자가 절대다수(84~97%) → 신규/주말 같은 범주형은 군집의 주축이 아님. 군집은 연속형 행동(세션 깊이)으로 갈림.
  • 약한 경향: 신규 비율 = 탐색(15.9%) > 고관여(10.2%) > 즉시이탈(3.3%). 즉시이탈은 거의 전부 재방문자(습관적으로 잠깐 들름).
  • 주말 비율 = 고관여(27.5%) 최고, 즉시이탈(18.3%) 최저. 다만 ±5%p로 결정적이지 않음.

7. PCA 2D 시각화

  • 2개 주성분이 전체 분산의 54%만 설명 → 2D 겹침은 정보 손실 탓(군집은 11차원에서 수행).
  • PC1이 사실상 '세션 깊이·관여도' 축: 즉시이탈은 한쪽 끝, 고관여는 반대 끝.
  • 일반탐색이 중앙에 넓게 퍼져 고관여로 연속적으로 이어짐 → 관여도는 연속 스펙트럼, K-Means가 이를 3구간으로 끊은 것.

8. 성능 평가는 어떻게 하나? (중요)

분류(DT/RF/KNN)처럼 정답 라벨이 없으므로 단일 정확도 숫자가 없다. 대신 3축으로 본다.

(1) 내부 지표 — 라벨 없이 군집 구조 자체를 본다

군집이 "얼마나 똘똘 뭉치고(응집) 서로 떨어졌나(분리)" 를 측정.

📖 지표 용어 풀이 — 셋 다 같은 질문("응집 + 분리")을 다르게 점수화. 단독 정답은 없고 함께 본다.
  • Silhouette(실루엣) — 점이 자기 군집엔 가깝고 옆 군집엔 먼가? 각 점마다 (b−a)/max(a,b) (a=자기 군집 평균거리, b=가장 가까운 다른 군집 평균거리). −1~1, 높을수록 좋음. 1=푹 박힘, 0=경계, 음수=잘못 묶임. (0.32 = 구조는 있으나 경계 약간 겹침)
  • Davies-Bouldin — 가장 닮은(겹치는) 두 군집이 얼마나 헷갈리나? (내 퍼짐 + 이웃 퍼짐) / (두 중심 거리)의 평균. 군집이 작고 단단하며 멀수록 작아짐. 낮을수록 좋음(0=이상).
  • Calinski-Harabasz — 군집 간 분산 / 군집 내 분산 비율. 서로 멀고 각자 뭉칠수록 큼. 높을수록 좋음. 절대기준 없이 K끼리 비교용.
지표 방향 현재/비고
Silhouette 높을수록 ↑ (−1~1) K=3 ≈ 0.32 — 약~중간. 행동 데이터는 경계가 흐려 0.3대가 흔함
Davies-Bouldin 낮을수록 ↑ 추가 권장. 군집 간 유사도(겹침) 척도
Calinski-Harabasz 높을수록 ↑ 추가 권장. (군집간 분산 / 군집내 분산) 비
Inertia(엘보우) 참고용 K 선택 보조일 뿐, 절대 성능 아님 
from sklearn.metrics import davies_bouldin_score, calinski_harabasz_score
print("Silhouette     :", silhouette_score(Z, cluster_labels))   # 높을수록
print("Davies-Bouldin :", davies_bouldin_score(Z, cluster_labels)) # 낮을수록
print("Calinski-Hara. :", calinski_harabasz_score(Z, cluster_labels)) # 높을수록

실루엣 0.32는 "구조는 있으나 군집 경계가 일부 겹친다"는 뜻. 행동 세분화에선 정상 범위이고, PCA(54%)에서 본 연속 스펙트럼과도 일치한다. 0.5+면 뚜렷, 0.2 미만이면 군집 근거 약함.

(2) 외부 검증 — 이 프로젝트의 핵심 평가

군집 학습에 Revenue를 쓰지 않았으므로, 사후에 군집별 구매율이 뚜렷이 갈리는지가 사실상의 성능 평가다. 현재 0.4% → 15% → 24% 단조 상승 → 행동 군집이 구매 성향을 잘 포착했다는 증거. 정량화하려면 군집-구매율 차이의 유의성을 검정한다.

from scipy.stats import chi2_contingency
chi2, p, *_ = chi2_contingency(pd.crosstab(profile['cluster'], profile['Revenue']))
print(f"chi2={chi2:.1f}, p={p:.2e}")   # p가 매우 작으면 군집별 구매율 차이가 유의

(외부 정답 라벨이 따로 있다면 ARI, NMI, V-measure로 군집-라벨 일치도를 잴 수도 있다.)

(3) 안정성 — 재현되는가

K-Means는 초기값에 민감하므로 여러 seed로 재실행해 배정이 일관적인지 본다.

from sklearn.metrics import adjusted_rand_score
base = KMeans(3, random_state=0, n_init=10).fit_predict(Z)
alt  = KMeans(3, random_state=99, n_init=10).fit_predict(Z)
print("ARI(run 간 일치도):", adjusted_rand_score(base, alt))  # 1에 가까울수록 안정

정리

"잘 분리됐나(내부 지표)" + "의미 있는 외부 변수와 맞나(외부=구매율)" + "재현되나(안정성)" 를 함께 봐야 한다. 이 노트북은 실루엣+엘보우(내부)와 구매율 단조성(외부)을 이미 쓰고 있고, 보강하려면 Davies-Bouldin·Calinski-Harabasz·카이제곱·ARI를 추가하면 된다.
