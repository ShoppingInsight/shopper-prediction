Random Forest 구매 예측 모델 — 설명 문서

한 문장 요약 — F1 기준으로 튜닝한 Random Forest는 raw test에서 F1 0.4348, Recall 0.6675를 기록해 default 모델보다 구매자를 훨씬 넓게 잡았다. K-Means로 요약한 방문 행동 유형을 추가 변수로 반영했을 때의 변화도 함께 살폈다.

개요

분석 질문

이 문서의 핵심 질문은 "Random Forest로 방문자의 구매 여부를 얼마나 잘 예측할 수 있고, 어떤 튜닝과 변수가 성능에 영향을 주는가"이다.

최종 평가는 극단값을 제거하지 않은 raw test를 기준으로 한다. IQR 방식으로 이상치를 제거하면 train 33.83%, test 33.31%가 빠지므로, 성능을 과대평가할 수 있는 주 평가 기준으로 쓰기보다 민감도 확인용으로 보는 편이 안전하다.

결론부터 — F1 기준으로 튜닝한 Random Forest가 구매 후보를 넓게 잡는 목적에 가장 잘 맞다. default 모델보다 오탐은 늘지만 실제 구매자를 훨씬 많이 잡아낸다.

이 문서 읽는 법

Random Forest

여러 Decision Tree를 만들어 투표하듯 분류한다. 단일 트리보다 안정적이고 변수 중요도를 볼 수 있지만, 튜닝을 잘못하면 train 데이터의 우연한 규칙까지 외울 수 있다.

K-Means

정답인 Revenue를 보지 않고 행동이 비슷한 세션을 묶는다. 여기서는 방문 세션을 3개 행동 유형으로 나누고, 그 유형 정보를 RF 입력 변수로 추가했다.

군집 변수 추가

RF 예측값과 K-Means 예측값을 평균낸 것이 아니다. rf_cluster_0 같은 군집 라벨 또는 중심거리 컬럼을 RF 입력 변수에 붙여 비교했다.

왜 raw test를 기준으로 보나

IQR 제거는 극단 세션을 줄여 모델을 안정적으로 보이게 할 수 있지만, 실제 쇼핑몰 로그에는 길게 탐색하거나 거의 바로 나가는 세션이 자연스럽게 존재한다. 그래서 최종 모델 비교는 원래 분포를 더 잘 대표하는 raw test에서 한다.

PageValues는 구매 이후 정보가 섞일 수 있는 누수 변수이므로, 데이터에 남아 있으면 제거하는 방어 로직을 둔다. 이번 실행 결과에서는 Random Forest 입력 CSV에 이미 포함되어 있지 않았다.

튜닝 이유: Accuracy만 높이면 안 된다

구매 비율은 약 15.6%로 낮다. 그래서 대부분을 "비구매"로 예측해도 Accuracy는 꽤 높게 보일 수 있다. 이 분석에서는 구매 후보를 실제로 찾아내는 목적이 중요하므로, Accuracy보다 Recall, F1, PR-AUC를 함께 본다.

validation F1은 train 데이터를 다시 나눈 내부 validation split에서 계산한 F1이다. 최종 test를 건드리지 않고도 트리 깊이처럼 과적합에 민감한 설정을 비교할 수 있게 해 주며, 구매/비구매가 불균형한 데이터에서 Precision과 Recall 중 한쪽만 좋아지는 모델을 거르기 위한 기준으로 썼다.

K-Means 군집 변수 추가

군집 변수는 models/clustering/csv/X_train_knn.csv, models/clustering/csv/X_test_knn.csv의 11개 행동 변수를 기준으로 만들었다. 누수를 막기 위해 scaler와 K-Means는 train 데이터에만 fit했고, test 데이터는 같은 변환을 적용한 뒤 predict와 transform만 수행했다.

1

기존 split 유지

RF의 X_train, X_test, y_train, y_test를 그대로 썼다. 비교 조건을 바꾸지 않기 위해서다.

2

train으로만 K-Means 학습

방문 행동 변수 11개를 log 변환하고 표준화한 뒤 KMeans(n_clusters=3)를 train에만 fit했다.

3

군집 라벨 추가

세션이 속한 군집 번호를 rf_cluster_0, rf_cluster_1, rf_cluster_2 one-hot 컬럼으로 붙였다.

4

중심거리 실험

각 세션이 세 군집 중심에서 얼마나 떨어졌는지도 rf_cluster_dist_*로 추가해 별도 비교했다.

결론 / 활용 방향

  • 결론: raw test 기준으로는 Random Forest tuned by F1이 구매 후보를 넓게 잡는 목적에 가장 잘 맞는다. Accuracy는 default보다 낮지만, 실제 구매자 382명 중 255명을 잡아 Recall과 F1이 크게 개선됐다.
  • 해석: 핵심 신호는 ExitRates, ProductRelated_Duration, total_duration이다.
  • 운영 관점: 구매 가능 고객을 넓게 모아 리마케팅 후보를 만들려면 F1/Recall 중심 모델이 맞다. 캠페인 단가가 높아 오탐 비용이 크다면 Precision 튜닝 모델을 별도 후보로 둘 수 있다.
  • K-Means 군집 변수: one-hot 군집 라벨은 F1 0.4348 → 0.4403, Recall 0.6675 → 0.6859로 소폭 개선을 보였다.
