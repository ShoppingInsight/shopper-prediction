01

핵심 요약

Best Raw Test F1 0.4403 RF + KMeans one-hot
Purchase Recall 0.6859 RF + KMeans one-hot
Best Precision 0.4026 Random Forest default
Best PR-AUC 0.3726 F1 튜닝 RF

이 리포트의 질문은 단순하다. "방문자가 구매할지 예측할 때, Random Forest만 쓰는 것보다 K-Means로 만든 방문자 유형 정보를 함께 넣으면 도움이 되는가?"를 확인한다. 최종 비교 기준은 극단값을 제거하지 않은 raw test 데이터다. IQR 제거 데이터는 표본을 약 33% 줄이므로 메인 평가가 아니라 민감도 분석으로만 사용했다.

결론부터 읽으면

models/clustering/csv/X_train_knn.csv로만 K-Means를 fit하고 cluster label을 one-hot feature로 추가했을 때 F1은 0.4348에서 0.4403, Recall은 0.6675에서 0.6859, ROC-AUC는 0.7871에서 0.7897로 소폭 개선됐다. 즉, 군집 정보는 큰 성능 향상까지는 아니지만 행동 유형을 요약하는 보조 신호로 작동했다.

cluster 중심까지의 거리 3개를 추가한 모델은 거리 feature 중요도는 높게 잡혔지만 F1과 Recall이 baseline보다 낮았다. 이번 실행 결과에서는 RF + KMeans one-hot이 가장 납득 가능한 융합 모델이다.

02

이 리포트 읽는 법

Random Forest

여러 개의 Decision Tree를 만들어 투표하듯 예측하는 지도학습 모델이다. 여기서는 Revenue, 즉 구매 여부를 맞히는 분류 모델로 사용했다.

K-Means

정답을 보지 않고 행동 패턴이 비슷한 방문 세션끼리 묶는 비지도학습 모델이다. 여기서는 방문자를 3개 유형으로 나누고, 그 유형 번호를 RF의 추가 입력으로 넣었다.

모델 융합

두 모델의 예측값을 평균낸 것이 아니다. K-Means가 만든 cluster label과 cluster distance를 새로운 컬럼으로 만들어 Random Forest 입력 feature에 추가한 것이다.

왜 이런 실험을 했나

Random Forest는 개별 변수의 조합을 보고 구매 여부를 예측한다. 예를 들어 ExitRates, ProductRelated_Duration, total_duration 같은 값을 직접 본다. 그런데 K-Means는 여러 행동 변수를 한꺼번에 보고 "이 방문자는 어떤 행동 유형인가"를 먼저 요약한다.

따라서 cluster feature를 추가한다는 것은 RF에게 "이 세션은 저관여/일반탐색/고관여 같은 행동 그룹 중 어디에 가까운가"라는 요약 정보를 더 알려주는 것이다. 이 요약 정보가 기존 변수만으로는 부족했던 패턴을 보완하면 성능이 좋아지고, 이미 기존 변수로 충분히 설명된다면 성능 변화가 작거나 오히려 떨어질 수 있다.

04

과적합과 하이퍼파라미터 튜닝

여기서 튜닝한 이유

Random Forest는 트리 깊이, 트리 개수, 분기 기준, leaf 크기 같은 설정에 따라 구매자를 너무 적게 잡거나 train 데이터에 과하게 맞을 수 있다. 이 분석에서는 불균형 데이터에서 Accuracy만 높이는 모델을 피하기 위해 F1과 Precision 기준 튜닝을 따로 비교했다.

특히 트리가 깊어질수록 데이터가 더 잘게 쪼개진다. 초반 분기는 많은 샘플이 공유하는 큰 패턴을 잡지만, 깊은 분기는 몇 개 안 되는 샘플의 우연한 조합까지 규칙처럼 외울 수 있다. 이 경우 train 점수는 계속 좋아지지만 처음 보는 validation/test 데이터에서는 같은 규칙이 재현되지 않아 성능이 떨어진다.

그래서 train 데이터 안에서 다시 stratified validation split을 만들었다. 이 분할은 최종 test를 대체하는 것이 아니라, 최종 test를 반복해서 들여다보며 모델을 고르는 일을 막기 위한 내부 점검용이다. stratify를 사용했기 때문에 구매/비구매 비율은 train과 validation에 비슷하게 유지된다.

06

K-Means 군집 feature 융합 실험

군집 feature는 models/clustering/csv/X_train_knn.csv, models/clustering/csv/X_test_knn.csv의 11개 행동 변수를 기준으로 만들었다. 누수 방지를 위해 scaler와 K-Means는 train 데이터에만 fit했고, test 데이터는 predict와 transform만 수행했다. 즉 test 데이터는 K-Means 중심을 만드는 데 전혀 쓰지 않았다.

1

기존 split 유지

Random Forest가 쓰던 X_train, X_test, y_train, y_test를 그대로 사용했다. 비교 조건을 바꾸지 않기 위해서다.

2

train으로만 K-Means 학습

models/clustering/csv/X_train_knn.csv의 방문 행동 변수 11개를 log 변환하고 표준화한 뒤, train 데이터로만 KMeans(n_clusters=3)를 fit했다. X_test_knn.csv는 같은 기준으로 변환한 뒤 predict만 수행했다.

3

cluster label을 RF 입력에 추가

각 세션이 속한 cluster 번호를 rf_cluster_0, rf_cluster_1, rf_cluster_2 one-hot 컬럼으로 바꿔 RF feature에 붙였다.

4

거리 feature도 별도 실험

각 세션이 세 군집 중심에서 얼마나 떨어져 있는지 rf_cluster_dist_0, rf_cluster_dist_1, rf_cluster_dist_2로 추가한 모델도 따로 비교했다.

실제로 RF에 추가된 feature

기존 RF는 방문 페이지 수, 체류 시간, 이탈률, 월, 방문자 유형 같은 원래 feature만 사용했다. 융합 모델에서는 여기에 K-Means가 만든 군집 정보를 추가했다. 예를 들어 어떤 세션이 cluster 2에 속하면 rf_cluster_2=1, rf_cluster_0=0, rf_cluster_1=0처럼 표시된다.

두 번째 실험에서는 one-hot label에 더해 각 세션이 세 군집 중심에서 얼마나 떨어져 있는지도 rf_cluster_dist_0, rf_cluster_dist_1, rf_cluster_dist_2로 붙였다. label은 "어느 군집인가"를 알려주고, distance는 "그 군집 패턴에 얼마나 전형적으로 가까운가"를 알려준다.

어떤 부분이 발전했나

cluster one-hot 모델의 개선 폭은 크지 않다. 따라서 "K-Means를 붙이면 RF가 크게 좋아진다"는 결론은 아니다. 더 정확한 해석은 "방문자 행동 유형을 나타내는 cluster label이 RF에 약간의 추가 정보를 제공했다"이다. baseline 대비 Recall은 0.6675에서 0.6859, F1은 0.4348에서 0.4403으로 올라갔다.

발전한 점은 RF가 개별 변수만 보던 상태에서 "행동 유형"이라는 요약 정보를 함께 보게 됐다는 것이다. cluster 2처럼 구매율이 거의 없는 군집과 cluster 1처럼 구매율이 높은 군집이 구분되면, RF는 비슷한 ExitRates나 체류 시간을 가진 세션 사이에서도 행동 맥락을 조금 더 세밀하게 나눌 수 있다.

distance feature는 세 군집 중심까지의 연속적인 거리 정보를 주면서 모델을 더 복잡하게 만들었다. 이 정보는 feature importance에서는 크게 나타났지만, Recall은 0.6597, F1은 0.4278로 baseline보다 낮았다. 따라서 이번 결과에서는 거리값보다 one-hot cluster label만 추가한 모델을 더 합리적인 융합 후보로 본다.

모델 해석과 활용 방향

이탈 지표가 가장 강한 신호다

ExitRates, BounceRates가 두 모델 모두 상위권에 있다. 세션이 끝나는 지점과 즉시 이탈하는 행동이 구매 예측에서 핵심 분기 역할을 한다.

체류 시간은 구매 의도와 연결된다

total_duration, ProductRelated_Duration, avg_time_per_product가 중요하게 나타난다. 상품 관련 페이지에서 충분히 머무는 세션은 구매 가능성이 높아진다.

운영 목적에 따라 모델을 선택해야 한다

구매 가능 고객을 넓게 잡아 리마케팅 후보를 만들려면 F1 튜닝 모델이 적합하다. 캠페인 비용이 높아 오탐을 줄여야 하면 Precision 튜닝 모델을 별도 후보로 볼 수 있다.

군집 정보는 보조 신호로 쓰는 게 맞다

K-Means one-hot feature는 Recall, F1, ROC-AUC를 소폭 개선했다. 그래도 핵심 변수는 여전히 ExitRates, ProductRelated_Duration, total_duration이므로, 군집은 주 모델을 대체하기보다 행동 유형을 요약하는 보조 feature로 해석하는 편이 안전하다.
