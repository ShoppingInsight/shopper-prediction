Technical Report

의사결정나무(Decision Tree) 모델 구현 및 기술 분석 보고서

의사결정나무 모델 파이프라인의 라인별 코드 구현 검증 및 불균형 클래스 처리를 위한 수리적 분석 보고서

개요 및 핵심 알고리즘 이론

본 보고서는 쇼핑객 구매 의사 예측 모델을 위해 구현된 의사결정나무 알고리즘의 세부 코드 동작 방식과 모델 평가 지표에 대한 수리적 배경을 명시합니다. 모델의 예측 신뢰도 확보를 위한 핵심 알고리즘 요소인 지니 불순도, 클래스 불균형 해결 기법 및 하이퍼파라미터 최적화(GridSearchCV)의 상세 구조는 다음과 같습니다.

1. 의사결정나무 알고리즘의 정의

의사결정나무는 입력 데이터의 다양한 독립 변수(Feature)들을 기준으로 정보를 분할하여 최종적인 종속 변수(Class)를 판별해나가는 지도학습 기반의 분류 알고리즘입니다. 정보 획득량(Information Gain)을 극대화하는 정보 분할 규칙(Decision Rules)을 제공하므로, 예측 결과에 대한 기여도를 직관적으로 판독할 수 있습니다.

2. 지니 불순도(Gini Impurity) 수리적 기준

지니 불순도는 노드 내 데이터 집합의 분류 다양성(혼잡도)을 평가하는 수리적 척도입니다. 특정 노드에서 추출된 샘플이 임의로 분류되었을 때 잘못 분류될 확률을 의미하며, 완전히 단일 클래스만 존재하는 상태(Impurity = 0)부터 반반씩 균등하게 혼재된 상태(Impurity = 0.5) 사이의 값을 지닙니다. 알고리즘은 분할 전후의 지니 불순도가 가장 크게 감소하는 방향으로 분기점을 설정합니다.

3. 클래스 불균형(Class Imbalance) 해소 방안

쇼핑객 행동 데이터셋 내 실제 구매자(1)의 비율은 약 16%, 비구매자(0)는 약 84%로 심각한 불균형 분포를 보입니다. 단순 정확도만을 극대화하려는 경향을 배제하고 희소 타겟 데이터(구매군)에 대한 감지력(Recall)을 향상시키기 위하여 손실 함수 계산 시 구매 데이터에 패널티 가중치를 배정하는 비용 민감 학습(Cost-sensitive Learning) 방식을 적용합니다.

4. 하이퍼파라미터 및 교차검증(CV) 최적화

하이퍼파라미터는 모델의 학습 용량 및 노드 확장 범위를 강제하기 위해 사전 설정하는 제어 변수입니다. 본 파이프라인에서는 max_depth(최대 깊이)와 class_weight(클래스 가중 비율)에 대해 교차검증(5-Fold Cross Validation)이 통합된 GridSearchCV 튜닝을 실행하여 일반화 성능이 최고조에 달하는 최적 매개변수를 확정합니다.

Python Library and Data Load 
import sys
import subprocess

# 의존 모듈 체크 및 예외 케이스 발생 시 동적 패키지 인스톨 수행
modules = ['pandas', 'numpy', 'matplotlib', 'seaborn', 'sklearn']
for mod in modules:
    try:
        __import__(mod)
    except ImportError:
        target_mod = 'scikit-learn' if mod == 'sklearn' else mod
        subprocess.check_call([sys.executable, "-m", "pip", "install", target_mod])

# 필요 라이브러리 및 하위 모듈 객체 가져오기
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.tree import DecisionTreeClassifier, export_text, plot_tree
from sklearn.model_selection import GridSearchCV
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score, confusion_matrix

# 전처리 파이프라인에서 추출된 학습 및 테스트용 데이터 로드
X_train = pd.read_csv('csv/X_train.csv')
X_test = pd.read_csv('csv/X_test.csv')
y_train = pd.read_csv('csv/y_train.csv').values.ravel()
y_test = pd.read_csv('csv/y_test.csv').values.ravel()

📌 구현 코드 기술 명세

  • 동적 모듈 임포트 루프: 개발 환경에 따라 특정 패키지가 누락되어 인터프리터 예외가 유발되는 상황을 복구하기 위해, 내부 subprocess 제어를 통해 실시간으로 패키지 매니저(pip)를 가동하여 환경 안정성을 극대화합니다.
  • values.ravel() 함수 적용 사유: read_csv() 함수는 기본적으로 데이터를 2차원 DataFrame으로 로드합니다. Scikit-learn의 ML 엔진은 1차원 타겟 어레이 형식을 수용하므로, ravel()을 사용하여 1차원 플랫 구조로 변환함으로써 차원 불일치(Dimension Mismatch) 경고 발생을 차단합니다.
⚙️ 2단계: GridSearchCV 하이퍼파라미터 최적화

과대적합(Overfitting) 제어를 위한 모델 깊이 임계값과 타겟 불균형 문제를 해소할 다차원 가중 비율을 정의하고, 지니 손실 함수 조건 하에서 교차검증을 통합 튜닝하는 과정입니다.

Python GridSearchCV Optimization 
# 1. 탐색할 하이퍼파라미터 그리드 딕셔너리 정의
param_grid = {
    'max_depth': [3, 4, 5, 6, 7, 8, 10, 12, 15],
    'class_weight': [
        None, 
        'balanced', 
        {0: 1, 1: 2}, 
        {0: 1, 1: 3}, 
        {0: 1, 1: 5}, 
        {0: 1, 1: 7}
    ],
    'criterion': ['gini']  # 평가 기준으로 지니 지수 채택
}

# 2. 난수 시드를 고정한 기본 DecisionTreeClassifier 생성
base_model = DecisionTreeClassifier(random_state=42)

# 3. F1-Score 최적화와 5-Fold 교차검증을 연계한 그리드 서치 구성
grid_search = GridSearchCV(
    estimator=base_model, 
    param_grid=param_grid, 
    scoring='f1',  # 정밀도와 재현율의 균형 척도인 F1-Score를 평가 기준으로 설정
    cv=5,          # 데이터를 5개 분할 영역으로 분할 검증
    n_jobs=-1,     # 가용한 전력 CPU 스레드를 병렬 연산에 활용
    verbose=1      
)

# 4. 파라미터 조합 훈련 실행 및 최종 최적 모델 적합
grid_search.fit(X_train, y_train)
best_model = grid_search.best_estimator_

📌 구현 코드 기술 명세

  • class_weight 비율 설정: 클래스 불균형에 대응하기 위해 수동으로 가중 비율을 정의합니다. {0: 1, 1: 3}은 타겟 클래스 1(구매)의 기여 비중을 타겟 클래스 0(비구매) 대비 3배로 높여 모델 가중 지니 지수를 보정합니다.
  • n_jobs=-1 매개변수: CPU 코어를 전부 병렬 스레드로 가동시키는 환경 설정 옵션입니다. 54개 조합에 대해 5배 교차 검증을 병렬 처리(총 270회 적합)하여, 순차 훈련 방식 대비 처리 연산 속도를 비약적으로 단축시킵니다.
📈 3단계: 예측 모델 성능 평가

훈련 완료된 최적 하이퍼파라미터 의사결정나무 모델을 검증용 데이터셋에 투입하여 주요 통계적 성능 지표를 검증 및 산출합니다.

Python Evaluation Metrics 
# 최적 가중 모델에 독립 테스트 변수 주입 및 예측값 도출
y_pred = best_model.predict(X_test)

# 수리적 지표 함수를 통한 연산 수행
accuracy = accuracy_score(y_test, y_pred)
precision = precision_score(y_test, y_pred)
recall = recall_score(y_test, y_pred)
f1 = f1_score(y_test, y_pred)

print(f"Accuracy : {accuracy:.4f}")
print(f"Precision: {precision:.4f}")
print(f"Recall   : {recall:.4f}")
print(f"F1-score : {f1:.4f}")

📌 성능 지표 수리적 정의

  • Accuracy (정확도): 전체 전체 테스트 케이스 대비 정확하게 참(TP+TN)을 판별한 비율입니다. $\frac{TP+TN}{TP+TN+FP+FN}$
  • Precision (정밀도): 모델이 참(1)으로 판별한 것 중 실제 원천 데이터가 참(1)인 비율입니다. 분모에 위양성(FP, 잘못 판정한 구매자)을 포함하여 마케팅 리소스 누수 정도를 파악합니다. $\frac{TP}{TP+FP}$
  • Recall (재현율): 실제 참(1) 중에서 모델이 누락하지 않고 식별해 낸 비율입니다. $\frac{TP}{TP+FN}$
  • F1-score: 정밀도와 재현율의 불균형을 고려한 조화평균(Harmonic Mean) 값입니다. $2 \times \frac{Precision \times Recall}{Precision + Recall}$

📌 시각화 기법 및 판독 기준

  • Confusion Matrix (혼동 행렬) 판독: 실제 참과 거짓에 대한 참양성(True Positive), 참음성(True Negative), 위양성(False Positive), 위음성(False Negative) 분포를 격자 히트맵으로 판독하여 정성 평가를 도모합니다.
  • Feature Importance (피처 중요도): 각 노드 분기 시 노드의 지니 불순도(Gini Impurity) 수치를 낮추는 데 기여한 가중 정보 이득(Information Gain) 평균값을 평가하여 독립 변수들의 기여 순위를 판별합니다.
의사결정 노드 내 가중치 적용 가치(Value)와 실제 샘플 수의 수리적 연계

클래스 가중치 적용에 따른 value 값의 계산식 검증

의사결정나무 모델 시각화 단계에서 노드 내부 속성값으로 표현되는 value = [2988.0, 3204.0]는 실제 데이터 샘플 개수와 불일치하며 소수점으로 정규화되는 현상을 보입니다. 이는 비용 민감 학습(Cost-sensitive Learning)에서 설정된 클래스 가중 비율(1:3)이 손실 평가 수식 내부로 가중 산출되어 나타난 수리적 특성입니다.

가중치가 포함된 의사결정나무 노드 가치 점수 연계 수식은 다음과 같이 귀결됩니다:

value = [ 실제 비구매(0) 샘플수 * 1.0, 실제 구매(1) 샘플수 * 3.0 ]

수치 검증 예시 (value = [2988.0, 3204.0] 인 분기 노드 기준):
• 실제 비구매 고객 수: 2,988명 ($2988.0 \div 1.0$)
• 실제 구매 고객 수: 1,068명 ($3204.0 \div 3.0$)
• 원본 데이터상 실제 구매자 비율: 26.33% ($1,068 \div (2,988 + 1,068)$)
해당 노드는 실제 인원 상 비구매자(2,988명)가 훨씬 다수를 차지하나, 모델이 가중치를 3배 적용한 결과 가치 점수가 구매 클래스(3204)가 비구매 클래스(2988)보다 크게 평가되므로 모델은 이 조건부 분기 하위에 도달하는 대상을 최종적으로 구매 가능성군(class: 1)으로 결정하여 리스크 관리를 수행합니다.

의사결정나무 모델 기반 비즈니스 인사이트 및 실행 전략

의사결정 규칙의 분기 경계와 변수 중요도 분석 결과를 종합하여 실무 지배적인 전략 의사결정 체계를 제언합니다.

1. 고객 이탈율(ExitRates) 3% 임계점 관리

최적화 모델의 루트 노드(Root Node)이자 최고 중요도 피처는 ExitRates(이탈률)입니다. 이탈률 수치가 3%를 초과하는 고객군은 타 변수 조건에 구애받지 않고 대부분 조기 이탈군(비구매)으로 할당됩니다.

■ 실행 전략: 페이지 사용성 허들 및 로딩 지연을 개선하여 이탈률 3% 이내 방어선을 구축하십시오.

2. 8분(479초) 체류 고객의 전환 촉진

이탈률이 3% 이내인 사용자 중, 제품 관련 상세 탐색 시간(ProductRelated_Duration)이 479초 이상인 고관여 집단의 실질 구매율은 26.33%에 육박합니다.

■ 실행 전략: 8분 임계 탐색 도달 유저군을 대상으로 실시간 챗봇 쿠폰 적용 등 프로모션 타겟 혜택을 부여하십시오.
