# 머신러닝 및 딥러닝 알고리즘 활용 정리

README.md의 `허용 기법`에 있는 알고리즘들이 각각 어떤 방법이고, 어떤 문제에 활용되는지 정리한 문서입니다.

---

## 회귀 알고리즘

회귀는 결과값이 숫자로 나오는 문제에 사용합니다. 예를 들어 가격, 점수, 수요량, 매출, 온도처럼 연속적인 값을 예측할 때 적합합니다.

| 알고리즘 | 어떤 알고리즘인가 | 주로 활용되는 곳 |
|---|---|---|
| 선형 회귀 | 하나의 입력 변수와 결과값 사이의 관계를 직선으로 설명하는 가장 기본적인 회귀 모델 | 공부 시간에 따른 점수 예측, 광고비에 따른 매출 예측, 온도 변화 예측 |
| 다중 선형 회귀 | 여러 입력 변수를 동시에 사용해 하나의 숫자 값을 예측하는 모델 | 집값 예측, 고객 소비 금액 예측, 상품 판매량 예측 |
| Ridge | 선형 회귀에 규제를 추가해 모델이 특정 변수에 과하게 의존하지 않도록 하는 모델 | 변수가 많은 데이터의 가격 예측, 수요 예측, 과적합을 줄여야 하는 회귀 문제 |
| Lasso | 중요하지 않은 변수의 영향력을 0에 가깝게 만들어 변수 선택 효과를 낼 수 있는 회귀 모델 | 많은 변수 중 중요한 요인 찾기, 설문 데이터 분석, 간단한 예측 모델 구성 |
| K-NN 회귀 | 예측하려는 데이터와 가까운 K개의 이웃 데이터를 찾아 평균값으로 예측하는 모델 | 비슷한 조건의 상품 가격 예측, 지역별 수요 예측, 유사 사례 기반 예측 |

---

## 분류 알고리즘

분류는 결과값이 정해진 범주 중 하나로 나오는 문제에 사용합니다. 예를 들어 스팸/정상, 합격/불합격, 고위험/저위험처럼 클래스를 예측할 때 적합합니다.

| 알고리즘 | 어떤 알고리즘인가 | 주로 활용되는 곳 |
|---|---|---|
| 로지스틱 회귀 | 결과가 특정 클래스에 속할 확률을 계산해 분류하는 모델. 이름은 회귀지만 주로 분류에 사용됨 | 스팸 메일 분류, 고객 이탈 예측, 질병 여부 예측, 합격/불합격 예측 |
| K-NN | 새로운 데이터와 가장 가까운 K개의 기존 데이터를 보고 다수결로 분류하는 모델 | 고객 유형 분류, 간단한 이미지 분류, 추천 시스템의 유사도 기반 분류 |
| Decision Tree | 조건을 따라가며 데이터를 나누는 나무 구조의 분류 모델 | 대출 승인 여부 예측, 구매 여부 예측, 규칙 기반 진단, 고객 세분화 |
| Random Forest | 여러 개의 Decision Tree를 만들고 결과를 종합해 분류하는 앙상블 모델 | 사기 거래 탐지, 고객 이탈 예측, 질병 위험 분류, 성능 비교용 기준 모델 |
| ANN/DNN | 여러 층의 인공신경망을 이용해 복잡한 패턴을 학습하는 딥러닝 모델 | 표 데이터 분류, 이미지 분류, 감성 분석, 사용자 행동 예측 |
| CNN | 이미지의 선, 모서리, 형태 같은 공간적 특징을 학습하는 딥러닝 모델 | 손글씨 숫자 인식, 제품 이미지 분류, 의료 영상 분석, 불량품 이미지 판별 |
| RNN | 이전 시점의 정보를 다음 예측에 반영하는 순차 데이터용 딥러닝 모델 | 주가나 날씨 같은 시계열 예측, 문장 분류, 감성 분석, 센서 데이터 이상 탐지 |

## 비지도 학습 알고리즘

비지도 학습은 정답 라벨이 없는 데이터에서 패턴이나 구조를 찾을 때 사용합니다. 데이터 탐색, 그룹화, 시각화에 자주 활용됩니다.

| 알고리즘 | 어떤 알고리즘인가 | 주로 활용되는 곳 |
|---|---|---|
| 군집화(Clustering) | 비슷한 데이터끼리 같은 그룹으로 묶는 방법 | 고객 세분화, 문서 그룹화, 지역 특성 분석, 사용자 행동 패턴 분석 |
| 차원 축소(PCA) | 많은 변수로 이루어진 데이터를 더 적은 수의 변수로 압축하는 방법 | 고차원 데이터 시각화, 모델 학습 전 전처리, 노이즈 감소, 변수 구조 파악 |

## 기타 기법

| 기법 | 어떤 기법인가 | 주로 활용되는 곳 |
|---|---|---|
| 하이퍼파라미터 튜닝 | 모델이 학습하기 전에 정해야 하는 설정값을 조정해 성능을 개선하는 과정 | K-NN의 K 값 선택, Decision Tree 깊이 조절, Random Forest 트리 개수 조절, 신경망 학습률 조절 |

## 프로젝트 주제에 맞춘 선택 기준

| 프로젝트 데이터/문제 형태 | 우선 고려할 알고리즘 |
|---|---|
| 숫자 값을 예측해야 한다 | 선형 회귀, 다중 선형 회귀, Ridge, Lasso, K-NN 회귀 |
| 두 개 이상의 범주로 분류해야 한다 | 로지스틱 회귀, K-NN, Decision Tree, Random Forest |
| 이미지 데이터를 다룬다 | CNN |
| 시간 순서가 있는 데이터를 다룬다 | RNN |
| 정답 라벨이 없다 | 군집화, PCA |
| 변수 개수가 많다 | Ridge, Lasso, PCA |
| 모델 결과를 쉽게 설명해야 한다 | 선형 회귀, 로지스틱 회귀, Decision Tree |
| 성능이 안정적인 비교 모델이 필요하다 | Random Forest |

## 보고서에 쓰기 좋은 설명 방식

보고서에서는 알고리즘 이름만 나열하기보다 다음 흐름으로 설명하는 것이 좋습니다.

1. 이 알고리즘이 어떤 방식으로 예측하거나 분류하는지 설명한다.
2. 우리 데이터의 어떤 특성 때문에 이 알고리즘을 선택했는지 적는다.
3. 같은 문제에 적용한 다른 모델과 성능을 비교한다.
4. 정확도뿐 아니라 장단점과 해석 가능성도 함께 설명한다.

예를 들어 분류 문제라면 `로지스틱 회귀`를 기본 모델로 사용하고, `Decision Tree`로 해석 가능성을 확인한 뒤, `Random Forest`로 성능 향상을 비교하는 식으로 구성할 수 있습니다.
